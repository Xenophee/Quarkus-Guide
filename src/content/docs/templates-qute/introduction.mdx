Pour construire des interfaces utilisateurs, plusieurs options sont possible aujourd'hui. La solution la plus répandue est d'utiliser un framework JavaScript côté client (React, Vue, Angular, etc.) pour construire une application web moderne qui demande beaucoup d'intéractivité et de gestion d'état côté front. Cependant, il existe des cas où une approche plus traditionnelle basée sur des templates côté serveur peut être plus appropriée, notamment pour des applications simples ou lorsque les performances sont une préoccupation majeure.

Pour ce faire, la plupart des frameworks proposent un moteur de template qui permet de générer du HTML côté serveur en utilisant des données dynamiques. Quarkus ne fait pas exception et propose une intégration avec le moteur **Qute**.


## Installation

Pour commencer avec les templates web de **Quarkus**, il faut ajouter les dépendances nécessaires à votre projet selon votre gestionnaire de dépendances. Il existe deux façon possible d'utiliser **Qute** dans votre projet : soit en utilisant `qute-web` qui va exposer directement les templates via HTTP, soit en utilisant `rest-qute` qui va vous permettre de rendre des templates depuis vos endpoints REST.

<Aside type="tip">
    `qute-web` est idéal pour les applications simples ou lorsque vous souhaitez exposer rapidement des templates sans avoir à écrire du code Java. En revanche, si vous avez besoin de plus de contrôle sur la logique métier ou si vous souhaitez intégrer des fonctionnalités plus complexes dans vos templates, `rest-qute` est la meilleure option. De plus, `rest-qute` vous permet de rendre des templates à partir de n'importe quel endpoint REST, ce qui offre une grande flexibilité pour construire des applications web plus complexes. De manière générale, `rest-qute` est recommandé pour la plupart des cas d'utilisation.
</Aside>


  <Tabs>
    <TabItem label="Maven" icon="seti:maven">

    Si vous utilisez **Maven** comme gestionnaire de dépendances, voici les dépendances à ajouter dans votre fichier `pom.xml` :

        Pour l'utilisation de `qute-web` :

        ```xml
        <dependency>
            <groupId>io.quarkiverse.qute.web</groupId>
            <artifactId>quarkus-qute-web</artifactId>
        </dependency>
        ```

        Pour l'utilisation de `rest-qute` :

        ```xml
        <dependency>
            <groupId>io.quarkus</groupId>
            <artifactId>quarkus-rest-qute</artifactId>
        </dependency>
        ```

    </TabItem>
    <TabItem label="Gradle" icon="seti:gradle">

    Si vous utilisez **Gradle**, ajoutez la dépendance suivante dans votre fichier `build.gradle.kts` :

        Pour l'utilisation de `qute-web` :

        ```kotlin
        implementation("io.quarkiverse.qute.web:quarkus-qute-web")
        ```

        Pour l'utilisation de `rest-qute` :

        ```kotlin
        implementation("io.quarkus:quarkus-rest-qute")
        ```

    </TabItem>

## Configuration des assets front

Par défaut, **Quarkus** sert les fichiers statiques (CSS, JavaScript, images, etc.) à partir du répertoire `src/main/resources/META-INF/resources`. Vous pouvez organiser vos assets dans ce répertoire pour les rendre accessibles depuis votre application. Par exemple, vous pouvez créer un sous-répertoire `css` pour vos fichiers CSS et un sous-répertoire `js` pour vos fichiers JavaScript. Par exemple, si vous avez un fichier `style.css` dans `src/main/resources/META-INF/resources/css`, il sera accessible via l'URL `/css/style.css` dans votre application.
