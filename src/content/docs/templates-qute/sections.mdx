Dans cette section, nous allons explorer les différentes structures de contrôle disponibles dans **Qute** pour gérer la logique de vos templates. Ces structures vous permettent d'ajouter des conditions, des boucles et des itérations à vos templates, ce qui est essentiel pour créer des interfaces utilisateur dynamiques et interactives.

## Conditions

Les conditions sont utilisées pour afficher ou masquer du contenu en fonction de certaines conditions.

### Définition de conditions

En **Qute**, vous pouvez utiliser la syntaxe suivante pour créer des conditions :

```html
{#if condition}
    <!-- Contenu à afficher si la condition est vraie -->
{/if}
```

Vous pouvez également ajouter une clause `else` pour gérer le cas où la condition est fausse :

```html
{#if condition}
    <!-- Contenu à afficher si la condition est vraie -->
{#else}
    <!-- Contenu à afficher si la condition est fausse -->
{/if}
```

Vous pouvez également utiliser des conditions imbriquées pour gérer des scénarios plus complexes :

```html
{#if condition1}
    <!-- Contenu à afficher si condition1 est vraie -->
{#else if condition2}
    <!-- Contenu à afficher si condition1 est fausse et condition2 est vraie -->
{#else}
    <!-- Contenu à afficher si condition1 et condition2 sont fausses -->
{/if}
```

Il est possible de faire des conditions avec les opérateurs suivants :

| Opérateur | Description |
|-----------|-------------|
| `==` | Égal à |
| `!=` | Différent de |
| `>` | Supérieur à |
| `<` | Inférieur à |
| `>=` | Supérieur ou égal à |
| `<=` | Inférieur ou égal à |
| `&&` | ET logique |
| `||` | OU logique |
| `!` | NON logique |


<Aside type="caution">
    Les opérateurs `>`, `<`, `>=` et `<=` ne sont pas supportés pour les valeurs `null`.
</Aside>


### Utiliser `when` ou `switch` pour les conditions multiples

Lorsque vous avez plusieurs conditions à évaluer, il peut être plus pratique d'utiliser la syntaxe `when` ou `switch` pour rendre votre code plus lisible. Voici comment vous pouvez utiliser ces structures en **Qute** :

```html
{#when name}
    {#is "Ripley"}
        <!-- Contenu à afficher si name est égal à "Ripley" -->
    {#is "Dallas"}
        <!-- Contenu à afficher si name est égal à "Dallas" -->
    {#else}
        <!-- Contenu à afficher si name ne correspond à aucun des cas précédents -->
{/when}
```

Les opérateurs supportés pour les conditions `when` sont les suivants :

| Opérateur | Description | Exemple |
|-----------|-------------| ----------|
| `==` | Égal à | `{#is "Ripley"}` |
| `!=` ou `not` | Différent de | `{#is not "Ripley"}` ou `{#case != "Ripley"}` |
| `>` | Supérieur à | `{#is > 10}` |
| `<` | Inférieur à | `{#is < 10}` |
| `>=` | Supérieur ou égal à | `{#is >= 10}` | |
| `<=` | Inférieur ou égal à | `{#is <= 10}` | |
| `in` | Appartient à une collection | `{#is in 'Ripley' 'Dallas'}` |
| `!in` | N'appartient pas à une collection | `{#is !in 'Ripley' 'Dallas'}` |

## Boucles et itérations

Les **boucles** sont utilisées pour itérer sur des **collections de données** et afficher du contenu pour chaque élément de la **collection**.

### Définition de boucles

En **Qute**, vous pouvez utiliser la syntaxe `each` suivante pour créer une **boucle** :

```html
{#each collection}
    <!-- Contenu à afficher pour chaque élément de la collection -->
{/each}
```

Vous pouvez aussi utiliser la syntaxe `for` pour itérer sur une **collection** et ainsi fournir un alias personnalisé (ici, `item`) pour chaque élément de la **collection** :

```html
{#for item in collection}
    <!-- Contenu à afficher pour chaque élément de la collection -->
{/for}
```

Lorsque vous n'avez aucun contenu à afficher pour une **collection** vide, vous pouvez utiliser la clause `else` pour gérer ce cas directement dans la boucle :

```html
{#for item in collection}
    <!-- Contenu à afficher pour chaque élément de la collection -->
{#else}
    <!-- Contenu à afficher si la collection est vide -->
{/for}
```

<Aside type="tip">
    L'usage de `each` ou `for` dépend de vos préférences et du contexte de votre template. `each` est plus concis et convient pour les cas simples, tandis que `for` offre plus de flexibilité en vous permettant de définir un alias personnalisé pour les éléments de la collection, ce qui peut améliorer la lisibilité de votre code.
</Aside>


### Expressions de boucle

**Qute** prend en charge plusieurs types d'**expressions de boucle** pour vous permettre de personnaliser la façon dont vous itérez sur vos données. Pour ce faire, vous devez utiliser l'élément de base sur lequel vous itérez, vous lui ajoutez un underscore `_`, puis le mot clef de **Qute**. Voici quelques exemples d'expressions de boucle que vous pouvez utiliser :

| Expression de boucle | Description |
|---------------------|-------------|
| `_count` | Retourne l'index de l'itération en cours, commençant à 1. |
| `_index` | Retourne l'index de l'itération en cours, commençant à 0. |
| `_isFirst` | Retourne `true` si l'itération en cours est la première de la collection, sinon `false`. |
| `_isLast` | Retourne `true` si l'itération en cours est la dernière de la collection, sinon `false`. |
| `_hasNext` | Retourne `true` s'il y a une itération suivante dans la collection, sinon `false`. |
| `_isEven` | Retourne `true` si l'index de l'itération en cours est pair, sinon `false`. |
| `_isOdd` | Retourne `true` si l'index de l'itération en cours est impair, sinon `false`. |
| `_indexParity` | Retourne "*even*" si l'index de l'itération en cours est pair, sinon "*odd*". |


Exemple concret d'utilisation de ces expressions de boucle :

```html
{#for item in collection}
    <p>Item {item_count}: {item.name}</p>
    {#if item_isFirst}
        <p>Ceci est le premier élément de la collection.</p>
    {/if}
    {#if item_isLast}
        <p>Ceci est le dernier élément de la collection.</p>
    {/if}
{/for}
```

Dans le cadre d'un `each`, vous pouvez utiliser le même concept en vous référant à l'élément de la collection en cours grâce au mot clef `it` :

```html
{#each collection}
    <p>Item {it_count}: {it.name}</p>
    {#if it_isFirst}
        <p>Ceci est le premier élément de la collection.</p>
    {/if}
    {#if it_isLast}
        <p>Ceci est le dernier élément de la collection.</p>
    {/if}
{/each}
```

<Aside type="tip">
    Les **expressions de boucle** sont très utiles pour ajouter des fonctionnalités supplémentaires à vos boucles, comme l'affichage d'un message spécial pour le premier ou le dernier élément de la collection, ou pour appliquer des styles différents en fonction de la parité de l'index.
</Aside>

## Définir des variables locales

Dans **Qute**, vous pouvez définir des variables locales à l'intérieur de vos templates pour stocker des valeurs temporaires ou pour simplifier l'accès à des données complexes. Pour ce faire, vous pouvez utiliser la syntaxe `let` ou `set` comme suit :

```html
{#let alien = "E.T"}
    <!-- Contenu du template où la variable est accessible -->
{/let}
```

Dans cet exemple, nous avons défini une variable locale `alien` avec la valeur `"E.T"`. Cette variable est accessible uniquement à l'intérieur du bloc `let`, ce qui vous permet de l'utiliser pour simplifier votre template ou pour stocker des valeurs intermédiaires.

<Aside type="tip">
    L'utilisation de variables locales peut améliorer la lisibilité de votre template en évitant les expressions complexes et en rendant votre code plus facile à comprendre. De plus, cela vous permet de réutiliser des valeurs calculées ou des données extraites sans avoir à les recalculer à chaque fois.
</Aside>

Il est possible également de définir une valeur par défaut pour une variable locale en utilisant la syntaxe suivante :

```html "?="
{#let alien ?= "E.T"}
    <!-- Contenu du template où la variable est accessible -->
{/let}
```

Dans cet exemple, la variable `alien` sera initialisée avec la valeur `"E.T"` uniquement si elle n'a pas déjà été définie auparavant. Si `alien` a déjà une valeur, cette valeur sera conservée et ne sera pas écrasée par la valeur par défaut.
